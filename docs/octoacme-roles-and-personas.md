# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

---

## QA Engineers

### Role Summary
QA Engineers focus on quality validation, test strategy, and acceptance testing. They ensure features meet acceptance criteria and quality standards before release.

### Responsibilities
- Define and execute test plans and test cases
- Validate features against acceptance criteria
- Identify and document bugs and regressions
- Maintain test environments and test data
- Collaborate with developers on quality improvements

### Goals
- Ensure high product quality and reliability
- Catch issues before they reach production
- Improve test coverage and automation

### Typical Communication
- Bug reports and test results
- Test plans and acceptance testing feedback
- Participation in sprint planning and demos

### Interaction with PM/PdM/Developers/QA
- **With Product Managers:** Reviews acceptance criteria and validates features meet business requirements
- **With Project Managers:** Reports on testing progress, risks, and blockers
- **With Developers:** Collaborates on test strategy, reports bugs, verifies fixes
- **With QA:** Coordinates test coverage and shares testing patterns

---

## UX Designer

### Role Summary
UX Designers design user flows, interfaces, and interactions to ensure usability, accessibility, and a cohesive user experience. They translate user needs and product requirements into intuitive designs.

### Responsibilities
- Create wireframes, mockups, and interactive prototypes
- Conduct user research and usability testing
- Define user flows and information architecture
- Ensure accessibility standards (WCAG) are met
- Maintain design systems and component libraries
- Collaborate on design reviews and iterate based on feedback

### Goals
- Deliver intuitive, user-centered designs
- Ensure consistent user experience across features
- Validate designs through user testing and data
- Reduce friction and improve user satisfaction

### Typical Communication
- Design specs, mockups, and prototypes
- User research findings and usability reports
- Design review sessions and feedback iterations
- Collaboration in sprint planning and demos

### Interaction with PM/PdM/Developers/QA
- **With Product Managers:** Collaborates on defining user needs, success metrics, and feature requirements; ensures designs align with product vision
- **With Project Managers:** Provides design timelines and dependencies; flags design risks early
- **With Developers:** Shares design specs and assets; partners on implementation feasibility and design handoff; clarifies design intent during development
- **With QA:** Defines expected UI behavior and acceptance criteria from a design perspective; validates that implementations match design specs

---

## DevOps Engineer

### Role Summary
DevOps Engineers maintain CI/CD pipelines, infrastructure, monitoring, and observability. They enable reliable, automated deployments and ensure system health.

### Responsibilities
- Build and maintain CI/CD pipelines and automation
- Manage infrastructure (cloud resources, Kubernetes, etc.)
- Implement monitoring, alerting, and observability
- Ensure security best practices in deployment and infrastructure
- Support incident response and troubleshooting
- Optimize build times, deployment processes, and resource usage

### Goals
- Ensure reliable, fast, and secure deployments
- Minimize downtime and deployment risks
- Improve system observability and incident response
- Automate repetitive operational tasks

### Typical Communication
- Deployment status and release notes
- Incident reports and postmortems
- Infrastructure and pipeline documentation
- Monitoring dashboards and alerts

### Interaction with PM/PdM/Developers/QA
- **With Product Managers:** Provides deployment timelines and infrastructure constraints; communicates system health and availability
- **With Project Managers:** Reports on deployment risks, infrastructure dependencies, and operational readiness
- **With Developers:** Collaborates on CI/CD pipeline improvements, build optimization, and debugging deployment issues; provides infrastructure support
- **With QA:** Maintains test environments and staging infrastructure; ensures CI runs automated tests reliably

---

## Technical Writer

### Role Summary
Technical Writers create and maintain living documentation, including user guides, API docs, release notes, and runbooks. They ensure documentation is accurate, up-to-date, and accessible.

### Responsibilities
- Write and maintain user-facing documentation
- Create API documentation and developer guides
- Draft release notes and changelog entries
- Maintain internal runbooks and operational docs
- Collaborate with engineers to understand features
- Ensure documentation follows style guides and standards

### Goals
- Provide clear, accurate, and useful documentation
- Reduce support burden through self-service docs
- Keep documentation in sync with product changes
- Improve documentation discoverability and searchability

### Typical Communication
- Documentation drafts and updates
- Release notes and changelog reviews
- Documentation feedback and improvement suggestions
- Collaboration in planning and release cycles

### Interaction with PM/PdM/Developers/QA
- **With Product Managers:** Understands feature intent and user value to create user-focused documentation; aligns docs with product messaging
- **With Project Managers:** Tracks documentation tasks in project plans; ensures docs are ready for release
- **With Developers:** Collaborates on technical accuracy; documents APIs, architecture, and implementation details; reviews code comments
- **With QA:** Uses test cases and acceptance criteria to inform documentation; validates documentation accuracy through testing
